## 🌐 網頁儀表板

`index.html` 透過 GitHub Pages 部署，讀取 `data.json` 自動更新，不需額外伺服器。

**版面區塊（由上至下）：**
1. 主燈號卡（價格、回檔進度、共振評分）
2. 技術指標（RSI、乖離率、MA、MACD）
3. 近期回檔機率
4. 子彈加碼建議
5. 子彈閒置狀況
6. 歷史回檔記錄

**資料狀態說明（網頁右上角指示燈）：**
- 🟢 盤中即時：交易時間（09:00–13:30）且 `data.updated` 在 30 分鐘內
- 灰色 休市・最後收盤：非交易時間且資料新鮮
- 🟡 資料延遲・X 分/小時/天前：時間戳 > 10 分鐘 **且** 處於異常狀態（交易時間內、`stale=true`、或 > 24 小時），最上方額外顯示黃色警示橫條。10 分鐘內視為新鮮，無條件不警告

## 🚀 部署教學（GitHub + Railway，完全免費）

### Step 1：建立 Discord Bot

1. 前往 [Discord Developer Portal](https://discord.com/developers/applications)
2. 點 `New Application` → 命名（例如：投資監控）
3. 左側選 `Bot` → `Reset Token` → 複製 Token（等下用）
4. 同頁面開啟 `Message Content Intent`
5. 左側選 `OAuth2 → URL Generator`
   - Scopes 勾：`bot` + `applications.commands`
   - Bot Permissions 勾：`Send Messages` + `Read Message History` + `View Channels`
6. 複製產生的 URL → 貌到瀏覽器 → 選你的伺服器 → 邀請機器人

### Step 2：建立 GitHub Repo

1. 前往 [github.com](https://github.com) → `New repository`
2. 命名（例如：`investment-bot`），設為 **Public**（GitHub Pages 需要）
3. 上傳所有檔案：`bot.py`、`requirements.txt`、`Dockerfile`、`Procfile`、`railway.json`、`index.html`

### Step 3：開啟 GitHub Pages

1. Repo 頁面 → `Settings` → `Pages`
2. Source 選 `Deploy from a branch`，Branch 選 `main`，資料夾選 `/ (root)`
3. 儲存後約 1 分鐘，網址 `https://<你的帳號>.github.io/<repo名稱>/` 即可訪問

### Step 4：建立 GitHub Personal Access Token

Bot 需要寫入 `data.json` 到 GitHub Repo，才能讓網頁顯示即時資料。

1. GitHub → 右上角頭像 → `Settings` → `Developer settings` → `Personal access tokens` → `Tokens (classic)`
2. `Generate new token (classic)` → 勾選 `repo` 權限 → 複製 Token

### Step 5：部署到 Railway

1. 前往 [railway.app](https://railway.app) → 用 GitHub 帳號登入
2. `New Project` → `Deploy from GitHub Repo` → 選你的 repo
3. 部署完成後，進入 `Variables` 頁面設定環境變數：

| 變數名稱 | 必填 | 預設 | 說明 |
|---------|------|------|------|
| `BOT_TOKEN` | ✅ | — | Discord Bot Token |
| `GITHUB_TOKEN` | ✅ | — | GitHub Personal Access Token（寫入 data.json 用）|
| `GITHUB_USER` | ⬜ | `znxuyz` | 若你 fork 到自己帳號，需改為自己的 GitHub 帳號 |
| `GITHUB_REPO` | ⬜ | `investment-bot` | 若 repo 名稱不同需修改 |
| `GITHUB_BRANCH` | ⬜ | `main` | 寫入 data.json 的目標分支 |

> ⚠️ 預設值寫死在 `bot.py` 開頭，自己 fork 後請務必確認 `GITHUB_USER` 與 `GITHUB_REPO`，否則 Bot 會把資料寫到原 repo（會失敗）。

4. 設定完成後 Railway 會自動重新部署
5. `Deployments` 頁面顯示 ✅ 即成功

> Railway 免費方案每月有 $5 額度，一個輕量 Discord Bot 大約跑 3~4 個月

### Step 6：設定 Discord 頻道

Bot 上線後，在你想接收通知的頻道輸入：
```
/設定頻道
```
Bot 會自動將該頻道設為日報和警報的接收頻道。

## ⏰ 自動排程

| 時間 | 動作 |
|------|------|
| 每日 09:00（週一至五） | 發送完整日報 |
| 每週一 09:00 | 發送週報 + 行動清單 |
| 每 13~17 分鐘（盤中 09:00–13:30） | 靜默偵測，觸發 −8/−15/−20% 才推播 |
| 每 5 分鐘（09:05–13:30，週一至五） | 更新網頁 data.json（盤中即時）|
| 每日 15:30（週一至五） | 收盤推送：更新 data.json + 推播收盤摘要 |
| 每月第一個平日 09:00 | 子彈閒置提醒（閒置 ≥90 天才發送；1 號為週末則順延到下週一）|
| Bot 啟動 / Railway 重啟 | 若當下在排程窗口內（09:05–13:30 或 15:30–15:59）才補推一次，其他時間跳過 |

> 國定假日自動跳過（透過 TWSE API 是否回傳即時報價判斷），無需手動設定假日清單

## 🛡️ 容錯與恢復

Bot 跑在 Railway 免費方案上，會遇到「定期重啟」與「TWSE API 偶發失效」兩種狀況。為此內建多層保護：

| 機制 | 行為 |
|------|------|
| **TWSE 月資料重試** | `fetch_monthly_twse` 對 5xx / `ConnectionError` / `Timeout` 自動重試 2 次（backoff 0.5s → 1s）；4xx 直接放棄不重試 |
| **GitHub data.json 預載** | Bot 啟動時 `load_data_json_from_github` 從 repo 摹現有 `data.json` 灌入 `_last_push_data`，重啟後第一次抓資料失敗仍能走 fallback |
| **日報 fallback** | 若 09:00 日報拿不到歷史資料，改用 `_last_push_data` 推「⚠️ 今日 TWSE 取資料失敗 ｜ 改顯示最後一次成功更新」訊息，並把 `stale=true` 推回 GitHub 讓網頁同步顯示警告 |
| **啟動推送排程闘門** | 啟動時的 force push 只在 09:05–13:30（盤中）或 15:30–15:59（收盤）才執行，避免夜間/週末重啟在 `data.json` 留下非排程時間戳 |
| **網頁過時偵測** | 網頁端自行檢查 `data.updated` 時間戳：≤10 分鐘新鮮（無警告，與 5 分鐘排程對齊）；>10 分鐘 + (交易時間 OR `stale=true` OR >24h) 才顯示黃色「資料延遲」警告 |

## 🚦 燈號說明

| 燈號 | 觸發條件 | 建議動作 |
|------|---------|---------|
| 🟢 正常持有 | 回檔 < 5% | 繼續每月定額，子彈保留 |
